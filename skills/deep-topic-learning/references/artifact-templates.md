# 学习沉淀模板（可复制）

按项目习惯放在每章末尾、或集中放在 `TROUBLESHOOTING.md`。

---

## 章节末尾：个人总结（Markdown）

```markdown
## 个人总结

- **本章核心**：用 2～4 条 bullet 写清「解决什么问题 + 关键机制」。
- **与上一章关系**：承接 / 对比哪张表或哪条数据流。
- **我修正过的理解**：（可选）之前错在哪、现在以什么为准（文档 / 源码 / 实验）。

*（可多次追加，建议标注日期）*
```

---

## 章节末尾：学习疑惑（单条 FAQ）

```markdown
### Q{n}: {简短标题 — 含关键词便于搜索}

**现象**：{报错、日志片段、或异常行为}

**曾误判**：{当时以为的原因}

**根因**：{最终确认的原因}

**验证**：{一条命令 / 最小脚本 / 预期输出}

**正确做法**：{配置项、版本范围、代码改法（若是项目内代码）}

**原则**：{可迁移的一条经验，例如依赖锁版本、REST 与 gRPC 端口区分}
```

---

## 项目级：TROUBLESHOOTING 条目（紧凑版）

```markdown
## {服务或库名} {版本可选}

| 现象 | 根因 | 处理 |
|------|------|------|
| ... | ... | ... |
```
